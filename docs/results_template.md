# Results

This file is a placeholder. The real version, `RESULTS.md`, is generated by the
pipeline from your actual run:

```bash
python -m ianoutage.build_report
```

`build_report` reads the peak table, the severity table, and the restoration
metrics and writes every figure straight from those files. It will not run, and
will not produce placeholder numbers, until the real pipeline outputs exist. That
is deliberate: the headline figures in your write-up can only come from data you
actually processed.

Fields the generated `RESULTS.md` will contain:

- statewide peak customers without power, and the timestamp it occurred;
- statewide restoration milestones (first below 50%, 25%, 10%, 5% of peak);
- top counties by peak customers out;
- top counties by customer-hours of outage (duration-weighted severity), with
  each county's restoration time to 10% of its own peak.

Do not fill these in by hand. Run the data.
